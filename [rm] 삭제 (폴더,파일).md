# 🗑️ rm (Remove) 명령어 치트시트

> 파일 또는 폴더를 **삭제**하는 리눅스/유닉스 명령어

---

## 📌 기본 문법

```
rm [옵션] [파일 또는 폴더...]
```

---

## ⚙️ 주요 옵션

| 옵션 | 설명 |
|------|------|
| `-r` | 폴더와 하위 내용을 재귀적으로 삭제 (recursive) |
| `-f` | 확인 없이 강제 삭제, 없는 파일도 오류 안 냄 (force) |
| `-i` | 삭제 전 파일마다 확인 메시지 표시 (interactive) |
| `-I` | 3개 이상 삭제 또는 재귀 삭제 시 1회만 확인 |
| `-v` | 삭제되는 파일 이름을 출력 (verbose) |
| `-d` | 빈 폴더 삭제 (rmdir 과 동일) |

---

## 💡 실전 예시

### 파일 삭제
```bash
# 파일 하나 삭제
rm file.txt

# 여러 파일 한 번에 삭제
rm file1.txt file2.txt file3.txt

# 특정 확장자 파일 모두 삭제
rm *.log
```

### 폴더 삭제
```bash
# 폴더와 하위 내용 모두 삭제
rm -r myfolder/

# 폴더 강제 삭제 (확인 없이)
rm -rf myfolder/

# 빈 폴더만 삭제
rm -d empty_folder/
# 또는
rmdir empty_folder/
```

### 옵션 활용
```bash
# 삭제 전 파일마다 확인
rm -i *.txt
# remove 'file1.txt'? y
# remove 'file2.txt'? n

# 삭제 과정 출력
rm -v *.log
# removed 'access.log'
# removed 'error.log'

# 없는 파일도 오류 없이 조용히 삭제
rm -f not_exist.txt
```

## ☠️ 위험한 명령어 주의

```bash
# ❌ 절대 실행 금지 - 시스템 전체 삭제
rm -rf /
rm -rf /*

# ❌ 경로 실수 주의 - 의도치 않은 전체 삭제
rm -rf $DIR/   # $DIR 이 비어있으면 rm -rf / 와 동일
```

> 💡 **Linux 커널 3.4+** 부터는 `rm -rf /` 실행 시 `--no-preserve-root` 옵션 없이는 거부됨
> 하지만 하위 경로 실수는 여전히 위험하므로 항상 경로를 먼저 확인할 것

---

## ⚠️ 주의사항

- `rm` 으로 삭제한 파일은 **휴지통 없이 즉시 삭제** → 복구 매우 어려움
- 와일드카드(`*`) 사용 시 반드시 **삭제 대상을 먼저 확인** (`ls *.log` 먼저 실행)
- 중요한 삭제 작업엔 `-i` 옵션 습관화 권장
- `rm -rf` 실행 전 경로를 `echo` 로 먼저 확인하는 습관 추천

```bash
# 삭제 전 대상 확인 습관
ls *.log          # 먼저 목록 확인
rm *.log          # 확인 후 삭제

# 경로 확인 후 삭제
echo $TARGET      # 경로 변수 먼저 확인
rm -rf $TARGET    # 확인 후 삭제
```

## 🔄 관련 명령어 비교

| 명령어 | 설명 |
|--------|------|
| `rm` | 파일/폴더 삭제 |
| `rmdir` | 빈 폴더만 삭제 |
| `trash-put` | 휴지통으로 이동 (별도 설치 필요) |
| `shred` | 파일 내용을 덮어쓴 후 삭제 (복구 방지) |
