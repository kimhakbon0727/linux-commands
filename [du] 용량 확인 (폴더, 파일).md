# 📂 du (Disk Usage) 명령어 치트시트

> 파일 또는 폴더가 **실제로 차지하는 디스크 용량**을 확인하는 리눅스/유닉스 명령어

---

## 📌 기본 문법

```
du [옵션] [파일 또는 디렉토리...]
```

---

## ⚙️ 주요 옵션

| 옵션 | 설명 |
|------|------|
| `-h` | 사람이 읽기 쉬운 단위로 출력 (KB, MB, GB) |
| `-s` | 합계만 출력 (summary) |
| `-a` | 파일까지 포함해서 전체 출력 |
| `-d` | 출력할 디렉토리 깊이 지정 (depth) |
| `-c` | 마지막에 전체 합계 출력 |
| `--max-depth` | 출력할 디렉토리 깊이 지정 (`-d` 와 동일) |
| `--exclude` | 특정 파일/폴더 제외 |
| `-x` | 현재 파일 시스템만 확인 (다른 마운트 제외) |

---

## 💡 실전 예시

### 기본 사용
```bash
# 현재 폴더의 용량 합계
du -sh .

# 특정 폴더 용량 확인
du -sh /var/log

# 여러 폴더 한 번에 확인
du -sh /var /home /tmp

# 전체 합계 포함
du -shc /var /home /tmp
```

### 하위 폴더별 용량 확인
```bash
# 1단계 하위 폴더까지만 출력
du -h --max-depth=1 /var

# 2단계까지 출력
du -h -d 2 /home

# 현재 디렉토리 하위 폴더 용량 전체 출력
du -h .
```

### 용량 큰 순서로 정렬
```bash
# 현재 디렉토리에서 용량 큰 순으로 정렬
du -sh * | sort -rh

# 특정 경로에서 상위 10개
du -sh /var/* | sort -rh | head -10

# 전체 시스템에서 가장 큰 폴더 찾기
du -h --max-depth=2 / 2>/dev/null | sort -rh | head -20
```

### 특정 항목 제외
```bash
# 특정 폴더 제외
du -sh /home --exclude="*.log"

# 여러 항목 제외
du -sh /var --exclude="cache" --exclude="tmp"

# 다른 마운트 포인트 제외 (현재 파티션만)
du -shx /
```

## 🔍 디스크 꽉 찼을 때 원인 찾기

```bash
# Step 1. 어느 파티션이 꽉 찼는지 확인
df -h

# Step 2. 해당 파티션에서 큰 폴더 찾기
du -h --max-depth=2 /var 2>/dev/null | sort -rh | head -20

# Step 3. 그 안에서 큰 파일 찾기
find /var/log -type f -size +100M

# Step 4. 오래된 로그 파일 확인
find /var/log -type f -name "*.log" -mtime +30
```

---

## 🔄 df vs du 비교

| 항목 | `df` | `du` |
|------|------|------|
| 확인 대상 | 파일 시스템 전체 | 특정 파일/폴더 |
| 용도 | 파티션 여유 공간 확인 | 어떤 폴더가 용량 차지하는지 확인 |
| 속도 | 빠름 | 느림 (파일 순회) |
| 삭제된 파일 | 포함됨 | 포함 안 됨 |

---

## ⚠️ 주의사항

- 루트(`/`)에서 실행 시 시간이 오래 걸릴 수 있음 → `--max-depth` 로 깊이 제한 권장
- 권한 없는 폴더는 접근 불가 → `2>/dev/null` 로 에러 메시지 숨김
- `df`와 `du` 결과가 다를 수 있음 → 삭제됐지만 프로세스가 열어둔 파일은 `df`에만 잡힘

```bash
# 에러 메시지 숨기고 실행
du -h --max-depth=1 / 2>/dev/null | sort -rh | head -20
```
