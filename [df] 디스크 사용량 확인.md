# 💾 df (Disk Free) 명령어 치트시트

> 파일 시스템의 **디스크 사용량과 여유 공간**을 확인하는 리눅스/유닉스 명령어

---

## 📌 기본 문법

```
df [옵션] [파일 또는 디렉토리...]
```

---

## ⚙️ 주요 옵션

| 옵션 | 설명 |
|------|------|
| `-h` | 사람이 읽기 쉬운 단위로 출력 (KB, MB, GB) |
| `-H` | 1000 단위로 출력 (1K=1000, `-h`는 1K=1024) |
| `-T` | 파일 시스템 타입 함께 출력 |
| `-t` | 특정 타입의 파일 시스템만 출력 |
| `-x` | 특정 타입의 파일 시스템 제외 |
| `-i` | 블록 대신 inode 사용량 출력 |
| `-a` | 모든 파일 시스템 출력 (가상 포함) |
| `--total` | 전체 합계 출력 |

---

## 💡 실전 예시

### 기본 사용
```bash
# 전체 디스크 사용량 (읽기 쉬운 단위)
df -h

# 파일 시스템 타입 포함 출력
df -hT

# 전체 합계 포함
df -h --total

# 특정 디렉토리가 속한 파티션 확인
df -h /var/log
```

### 파일 시스템 필터
```bash
# ext4 파일 시스템만 출력
df -t ext4

# tmpfs 제외하고 출력
df -hx tmpfs

# inode 사용량 확인 (파일 수가 많을 때 유용)
df -i
df -ih
```

### 출력 결과 해석
```bash
$ df -h
Filesystem      Size  Used Avail Use%  Mounted on
/dev/sda1        50G   20G   28G  42%  /
/dev/sdb1       200G  150G   44G  78%  /data
tmpfs           3.9G     0  3.9G   0%  /dev/shm
```

| 항목 | 설명 |
|------|------|
| `Filesystem` | 파일 시스템 (장치명) |
| `Size` | 전체 용량 |
| `Used` | 사용 중인 용량 |
| `Avail` | 사용 가능한 용량 |
| `Use%` | 사용률 |
| `Mounted on` | 마운트 위치 |

---

## 🔍 활용 팁

```bash
# 사용률 80% 이상인 파티션만 필터링
df -h | awk 'NR>1 && int($5) >= 80 {print}'

# 특정 마운트 포인트만 확인
df -h /home /var /tmp

# 디스크 꽉 찼을 때 — inode 확인 (용량은 남는데 파일 생성 안 될 때)
df -ih
```

---

## 🔄 df vs du 비교

| 항목 | `df` | `du` |
|------|------|------|
| 확인 대상 | 파일 시스템 전체 | 특정 파일/폴더 |
| 용도 | 파티션 여유 공간 확인 | 어떤 폴더가 용량 차지하는지 확인 |
| 속도 | 빠름 | 느림 (파일 순회) |

---

## ⚠️ 주의사항

- `df`는 **파일 시스템 단위**로 보여줌 → 특정 폴더의 용량이 궁금하면 `du` 사용
- 삭제된 파일이라도 프로세스가 열어두고 있으면 `df`에서 공간이 안 줄어들 수 있음 → 해당 프로세스 재시작 필요

```bash
# 삭제됐지만 점유 중인 파일 확인
lsof | grep deleted
```
